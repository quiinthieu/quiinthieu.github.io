Introduction

How can we prepare for storms before they hit? After completing the Google Data Analytics Professional Certificate, I wanted to dive into a real-world problem—analyzing U.S. storm events. By studying past storm data, we can uncover patterns that help improve disaster preparedness.

In this two-part blog series, I’ll guide you through the six phases of data analysis:

  1. Ask
  2. Prepare
  3. Process
  4. Analyze
  5. Share
  6. Act

Today, we’ll focus on the Ask, Prepare, and Process phases.

Ask Phase: Defining the Problem

Before diving into the data, I started by asking some key questions:

  • What types of storms are most common?
  • Where do storms frequently occur?
  • Which storms have the greatest impact in terms of damage and fatalities?

These questions would guide my entire analysis and help me identify the biggest threats from storms across the U.S. The goal was clear: to uncover insights that could support better disaster preparedness efforts.

Prepare Phase: Gathering the Data

To address these questions, I turned to the NOAA Storm Event Database. This dataset contains records of U.S. storm events from January 1950 to June 2024, detailing everything from location and event type to damage and fatalities.

The dataset has 51 columns, each loaded with important data. However, real-world datasets like this are often messy, containing missing or inconsistent values. I knew I’d have to clean it up before performing meaningful analysis.

Process Phase: Cleaning and Prepping for Analysis

With the raw data in hand, the next step was to clean it up. Here’s how I approached it:

  1. Removing Duplicates: I confirmed that there were no duplicate records—always a good starting point!
  2. Handling Missing Values: For columns like damage_property and damage_crops, I converted estimates like “10K” to actual numbers (e.g., $10,000) and imputed missing values as 0. This decision was based on the fact that missing values often indicate that no damage was reported.
  3. Tornado-Specific Fields: Fields like tor_f_scale (tornado scale) were only relevant for tornado events, so I left them as missing for non-tornado events.
  4. Data Type Conversion: I transformed date and location fields into appropriate formats, ensuring the data would be ready for analysis later.

What’s Next?

Now that the data is cleaned and ready, the next phase is to start analyzing it. In the second part of this blog series, I’ll walk you through how I explored storm patterns, their impacts, and what these insights tell us about future preparedness.

Want to dig deeper? Check out my notebook and ongoing progress on GitHub.

Stay tuned—there’s more to come!
